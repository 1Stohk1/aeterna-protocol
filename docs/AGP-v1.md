# AGP-v1 — Aeterna Genomic Protocol, version 1 (frozen)

**Status:** frozen by the Motore Scientifico orchestrator on 2026-04-20.
Breaking changes require bumping to `AGP-v2`.

The Aeterna Genomic Protocol is the on-wire format for every scientific
result produced by a Guardian. It is called "Genomic" for historical reasons
tied to Missione Alpha — the structure is general and carries any supported
task.

## 1. Design goals

1. **Bit-perfect reproducibility.** Two Guardians running the same task with
   the same seed and the same pinned dependencies must produce byte-identical
   `scientific_hash` values. Otherwise the consensus pipeline fails at
   Level 2.
2. **Auditable provenance.** Every field a block depends on is covered by
   `security.payload_hash`, which is signed by the Santuario.
3. **Compact gossip.** Payloads carry *metrics*, not full trajectories. If a
   peer wants the raw trace it issues a follow-up RPC.

## 2. Canonical schema

```json
{
  "header": {
    "protocol_version": "AGP-v1",
    "timestamp": 1713542400,
    "node_id": "Prometheus-0"
  },
  "payload": {
    "id_task": "TASK-a1b2c3d4-...",
    "tipo_analisi": "tumor_growth_gompertz",
    "parametri": { "...": "..." }
  },
  "reproducibility": {
    "seed_rng": 424242,
    "julia_version": "1.10.2",
    "package_manifest_hash": "sha256:a1b2c3d4..."
  },
  "results": {
    "metrics": { "N_final": 1.23e9, "doubling_days": 14.6, "...": "..." },
    "scientific_hash": "sha256:..."
  },
  "performance": {
    "execution_time_ms": 847.213
  },
  "security": {
    "payload_hash": "sha256:...",
    "results_scientific_hash": "sha256:...",
    "signature": "hex:dilithium5-signed-payload-hash",
    "public_key": "hex:dilithium5-public-key",
    "consensus_status": "PENDING",
    "pow_nonce": 194583,
    "pow_hash": "0000..."
  }
}
```

### Field-level contract

| Field | Type | Notes |
|---|---|---|
| `header.protocol_version` | string | MUST be exactly `"AGP-v1"` for this doc. |
| `header.timestamp` | int64 | Unix seconds, UTC, from the Sentinel at dispatch time. |
| `header.node_id` | string | The Guardian's public ID from `aeterna.toml`. |
| `payload.id_task` | string | UUID4 prefixed with `TASK-`. |
| `payload.tipo_analisi` | enum | One of the six locked task kinds (see §3). |
| `payload.parametri` | object | Task-specific, schema per §3. |
| `reproducibility.seed_rng` | int63 | **Generated by the Sentinel** and imposed on Julia. |
| `reproducibility.julia_version` | string | Reported by Julia at execution time. |
| `reproducibility.package_manifest_hash` | string | SHA-256 of the pinned Manifest.toml. |
| `results.metrics` | object | Float/int scalars only — no trajectories. |
| `results.scientific_hash` | string | `sha256(json.dumps(metrics, sorted))`. |
| `performance.execution_time_ms` | float | Engine wall-clock for the compute block, reported by Julia. **Informational** — deliberately NOT covered by `payload_hash` because wall-clock varies across hardware. Consumed by `PoUW / t_exec` in the Trust Score. |
| `security.payload_hash` | string | `sha256(canonical_json({header,payload,reproducibility,results}))` — note `performance` is excluded. |
| `security.signature` | string | Hex-encoded Dilithium-5 signed message over the 32-byte `payload_hash`. Receivers MUST verify it with `security.public_key` and MUST independently recompute `payload_hash` from the canonical payload before accepting the block. |
| `security.public_key` | string | Hex-encoded Dilithium-5 public key for the signing Guardian. |
| `security.pow_nonce` | int | Nonce that solves PoW difficulty. |
| `security.pow_hash` | string | `sha256(f"{payload_hash}:{nonce}")`, leading zeros ≥ difficulty. |
| `security.consensus_status` | enum | `PENDING` → `VALIDATED` → `FINALIZED` or `REJECTED`. |

## 3. Task kinds (v0.0.1, frozen)

The Sentinel dispatches exactly these six values via ZMQ to Julia. Any other
`tipo_analisi` value is a protocol violation and is rejected by `zmq_server.jl`.

| `tipo_analisi` | Julia function | Purpose |
|---|---|---|
| `genome_analysis` | `analizza_genoma` | Macro scan — GC, AT, TATA box, length |
| `genomic_entropy` | `calcola_entropia_genomica` | Shannon entropy of nucleotide distribution |
| `dna_mutation_hamming` | `calcola_mutazioni_dna` | Hamming-distance mutation profile (driver/passenger precursor) |
| `tumor_growth_gompertz` | `simula_crescita_tumorale` | Gompertz SDE tumor growth |
| `tumor_therapy_sde` | `simula_terapia_oncologica` | Gompertz SDE + pharmacokinetic drug pulse |
| `protein_folding_hp` | `avvia_folding_missione_alpha` | 2D HP-lattice Metropolis folding |

### Parameter schemas

- **genome_analysis / genomic_entropy** — `{ "sequence": "<ACGT...>" }`
- **dna_mutation_hamming** — `{ "ref": "<ACGT...>", "obs": "<ACGT...>" }` (equal length)
- **tumor_growth_gompertz** — `{ "N0": float, "rho": float, "K": float, "sigma": float, "days": int }`
- **tumor_therapy_sde** — `{ "N0": float, "rho": float, "K": float, "sigma": float, "days": int, "efficacia_farmaco": float, "giorno_inizio": int }`
- **protein_folding_hp** — `{ "sequence": "<HP...>", "steps": int }`

## 4. Reproducibility contract

The three `reproducibility` fields exist because Missione Alpha uses
stochastic integration (`dW_t` in Gompertz/Therapy SDE) and Monte Carlo
(HP lattice Metropolis). Without a pinned seed and a pinned Julia/Manifest
hash, two honest Guardians would produce *different* `scientific_hash`
values and the PoC layer would falsely mark the first one as adversarial.

**The Sentinel is the canonical source of `seed_rng`.** Julia receives it in
the request, calls `Random.seed!(seed)` at the entry of the handler, and
mirrors it back in the response. This design was chosen so the task
dispatcher centrally controls the experimental parameter space, and two
Guardians asked to validate the same AGP block can reproduce the exact same
trace on any machine.

The `package_manifest_hash` is the SHA-256 of the resolved `Manifest.toml`
of the `scientific/` Julia project. Mismatched hashes between two Guardians
are a hard consensus-pipeline failure — not a negotiation.

## 5. Versioning

`AGP-v1` is the first protocol version. Any change to:

- the set of task kinds,
- the schema of any task's `parametri`,
- the structure of `results.metrics`,
- the canonical hashing algorithm,

requires a version bump to `AGP-v2` and a migration plan. Guardians running
`v1` and `v2` must coexist on the network for at least 2 epochs (~20,000
blocks) before `v1` is retired.
